신규 계약 + BTF 물량 (6개월)
단순히 상점 수를 채우는 것이 아니라, 물량을 만들어내는 상점을 확보하는 것이 진짜 목표입니다. 아웃바운드 + 인바운드 듀얼 채널로 전환율을 높이되, 앵커 상점(고물량) 확보 전략을 병행하여 상점 당 물량 기여도를 극대화합니다.
Mission
이 로드맵이 이루려는 것
"상점을 많이 영업하는 것이 아니라,
물량을 만들어내는 상점을 효율적으로 확보하는 것"
상점 수 × 상점 당 물량 = 전체 물량 목표 달성. 둘 중 하나만으로는 안 됩니다.
목표 1: 상점 수 (Coverage)
10,000건
신규 계약 누적. 이탈을 감안한 순증 확보. 더 많은 상점이 부릉 네트워크에 올라올수록 라이더 효율이 높아지고 배달 품질이 올라감
목표 2: 물량 (Volume)
BTF 160만건
5대존 BTF 연간 목표 ~162만건 (경영계획). 소물량 상점 1,000개보다 앵커 상점 100개가 물량 기여도는 더 높을 수 있음
앵커 상점 전략: 왜 "누구를" 영업하느냐가 중요한가
소물량 상점
~100건/월
수는 많지만 물량 기여 낮음
앵커 상점
500건+/월
1개 = 소물량 5개 이상 효과
메가 상점
1,000건+/월
프랜차이즈급, 영업 난이도 높음
상점 10,000건을 모두 소물량(100건/월)으로 채우면 → 월 100만건. 목표에 한참 못 미침.
앵커 상점(500건+)을 20% 섞으면 → 소물량 8,000 × 100 + 앵커 2,000 × 500 = 월 180만건. 목표 초과.
앵커 상점 영업 전략
타겟팅: 플랫폼 다중 입점 상점, 월 주문 500건+ 예상, 현재 경쟁사(배민/요기요) 배달대행 이용 중
피칭 무기: 비용 절감 시뮬레이션이 강력 (월 수백만원 차이), SLA 비교 데이터, 묶음배송 효과
계약 조건: 할인 요금제 + 장기 계약으로 락인. 현장 즉시 계약 확정
우선순위: 리드 스코어링에 "예상 물량" 가중치 부여. 고물량 리드 → 영업사원 우선 배정
Problem & Levers
현재 문제와 5가지 성장 레버
문제 정의
1. 높은 이탈률: 12개월 잔존율 40~50%. 신규 확보 속도 > 이탈 속도여야 성장
2. 아웃바운드 한계: 영업사원 1인당 처리량에 물리적 한계. 사람만 늘리면 비효율
3. 인바운드 부재: 상점주가 관심 있어도 셀프 가입 경로 0%
4. 물량 무관 영업: 상점 수만 추적, 물량 기여도 측정 안됨. 소물량 위주 영업 리스크
5가지 성장 레버
1. 전환율 향상: 세일즈 툴로 현장 즉시 계약 (15% → 30%)
2. 커버리지 확대: 8개 권역 + 인력 30% 증원
3. 인바운드 채널: 프로덕트가 리드 생성 → 영업 마무리
4. 앵커 상점 집중: 고물량 상점 타겟팅으로 물량 목표 동시 달성
5. 이탈 방지: 온보딩 자동화 + 조기경보 (잔존율 65% → 75%)

Risks & Mitigations
주요 리스크와 대응
리스크영향대응
인바운드 배달 불가 지역 가입높음 즉시 이탈시뮬레이터에서 주소 입력 시 배달 가능 여부 자동 체크. 불가 시 "상담 신청"으로 전환
인바운드 리드 품질 낮음중간 전환율 하락사업자번호 검증 + 최소 정보 요구로 필터링. 영업사원 확인 단계 유지
권역 확장 시 라이더 부족높음 계약 후 이탈계약 전 권역별 라이더 수급 현황 확인 필수. 불가 권역 → 계약 보류
영업사원 증원 지연중간 목표 미달인바운드 비율 확대로 1인당 부담 분산. 보수적 시나리오 대비
포스 설치 거부중간 온보딩 이탈포스리스 옵션 정책 논의 (별도). 인바운드 상점은 디지털 친화적이라 거부율 낮을 것
소물량 상점 편중높음 상점 수 OK, 물량 미달리드 스코어링에 예상 물량 가중치 반영. 앵커 상점 비율 KPI (20%) 추적. 앵커 전용 피칭 자료
앵커 상점 이탈 시 물량 급감높음 물량 변동성앵커 전담 매니저 배정 + SLA 보장. 이탈 예측 조기경보를 앵커에 우선 적용. 포트폴리오 분산
앵커 상점 영업 난이도 높음중간 전환율 하락비용 절감 시뮬레이션 고도화 (실제 데이터 기반). 시범 배달 제안. 의사결정자 직접 미팅 지원
