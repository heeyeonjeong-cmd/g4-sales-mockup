Phase 1
기반 구축 + 현장 계약

  • 운영 영업 표준 프로세스 정립, 플레이북 v1.0 제작, 기존 권역 집중 영업
  • 마케팅 SLA 지표 시각화 피칭 자료, 디지털 리플렛, 경쟁사 비교 1-pager
  • 데이터 핵심 KPI 대시보드 (전환율, 리드타임), 전환 퍼널 추적 시작

Phase 2
권역 확장 + 효율화
6~7월 (2개월) | 목표: 누적 6,800건
  • 세일즈 툴 기가입/해지 상점 표시, 개인별 대시보드, 루트 최적화, SLA 피칭 뷰
  • 운영 서울 5개 + 부천/부산 권역 순차 오픈, 인력 +30% 채용/배치, 온보딩 프로그램
  • 마케팅 상점주 성공 사례 3건+, 경쟁사 전환 가이드, 지역별 맞춤 자료
  • 데이터 코호트 분석 자동화, 권역별 성과 비교, 이탈 패턴 분석
핵심 목표: 8개 권역 동시 가동 + 1인당 계약 수 1.5배 향상
Phase 3
인텔리전스 + 정점
8~9월 (2개월) | 목표: 누적 10,800건
  • 세일즈 툴 AI 리드 스코어링, 이탈 예측 모델, 자동 리마인드, 챗봇 FAQ
  • 운영 성과 기반 인센티브 가동, 영업사원 스케일업, 권역 미세 조정
  • 마케팅 인바운드 리드 채널 (웹/소셜), 상점주 추천 프로그램, 자동화 캠페인
  • 데이터 전환 가능성 예측, 이탈 확률 모델, 영업 ROI 측정 체계 완성

왜 4트랙이 필요한가?
툴만으로는 부족한 이유
시뮬레이션/계약 기능이 있어도 영업할 권역과 사람이 없으면 10,000건 불가. 현재 1개 권역 체제로는 물리적 한계.
운영이 뒷받침해야 하는 이유
8개 권역 동시 운영은 표준 프로세스 + 플레이북이 없으면 품질 편차 발생. 신규 권역 온보딩 속도 좌우.
피칭 자료가 전환율을 좌우
인터뷰에서 확인: 영업사원별 피칭 편차 큼. 표준화된 피칭 무기 (SLA, 성공사례)가 전환율 직결.
측정 없이 개선 없다
현재 전환율, 리드타임조차 정확히 모름. 측정 체계가 있어야 어떤 레버가 효과적인지 판단 가능.
